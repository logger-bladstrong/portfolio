// Author: Zirui Zhang
// Category: Strategy
// Status: Solved

Traditional programming is like teaching a computer to follow a recipe: "Add sugar, stir, bake." You define every rule explicitly.

Machine Learning (ML), however, is different. It's like giving the computer a thousand photos of cakes and asking it to figure out what a cake looks like on its own. It's not about explicit rules; it's about patterns.

The Paradigm Shift

When I wrote my first "Hello World" in Python, I felt like a commander giving orders. But when I ran my first Linear Regression model, I felt like a gardener planting seeds and watching them grow.

# A simple glimpse into Scikit-learn

from sklearn.linear_model import LinearRegression
import numpy as np

# Data: Hours studied vs. Score
X = np.array([[1], [2], [3], [4]])
y = np.array([10, 20, 30, 40])

model = LinearRegression()
model.fit(X, y)

print(model.predict([[5]])) # Output: [50.]

The magic happens in the .fit() function. The machine "learned" the relationship between X and y without me writing y = 10 * x.

This is just the beginning. From Supervised Learning to Neural Networks, the path ahead is foggy but fascinating.
